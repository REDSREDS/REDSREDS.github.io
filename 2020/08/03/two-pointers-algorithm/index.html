Two Pointers Algorithm

Posted on 2020-08-03

Two pointers algorithm is usually used to deal with problems involving arrays and lists. Usually move toward one direction and never go back.

1. Two direcitons

Two directions usually means one pointer at start, the other at end. those two pointers move towards each other. Problems like palindrome or two sum can be solved using this method.

56.Two Sum

Brute force way to solve the problem is to use two loops, O(N^2) time complexity. However, we know that, for a given number, when sum with the largest number in the array, the result is still smaller than target, then there is no need to calculate the sum with others for this number. Thus, it could be optimized with two pointers. The array should be sorted first. then, one pointer at the start position and one at the end. the left pointer only goes right and right only goes left. So the left pointer always points at the smallest, and right always points at the largest. If sum > target, right–, if sum < target, left++.

2. Same direction

This method usually involves two pointers at the start position, one goes faster than the other along the same direciton to the end of the array or linkedList.

521.Remove duplicates

Again, because the pointers will not go backward, we need to sort the array first. The slow pointer points to the first position of the duplicate, and the fast pointer look for the identical number. once found, swap value with the slow pointer.

Related questions:

610. Two Sum - Difference equals to target

This is different a typical two sum problem, which uses two direction pointer method. The reason to use same direction method is that, we don’t want any pointer to go backward.

3. Sliding window

604.Window sum

Two methods: 1) prefix sum: new array to store the prefix sum, then traverse the prefix sum to calculate the window sum. O(N). 2) sliding window: maintain an interval using two pointers, whenever move the window, add one from end and remove one from the start.

Related question:
1375. Substring With At Least K Distinct Characters: it is like sliding window, but the size of the interval changes. right pointer always points to the next position of the end of smallest substring, at specific left pointer.

Others

380. Intersection of Two Linked Lists: Two pointers, every time each pointer move 1 step, if two linked lists have the same length, then the two pointers will meet at the first intersection. What if lengths are different? First Method is to calculate the difference of the length first, let the pointer at the longer linkedlist moves the difference first, and then two pointer move at the same time until intersect. Second Method is for each pointer, move till end then goes to the head of the other list, so that both pointer moved along two lists, tail to head, and the first time they meet is the first intersection of the linkedlist.
102. Linked List Cycle: slow pointer moves 1 step while fast pointer moves 2 steps. Note the null pointer error.
103. Linked List Cycle II: need some calculation, basically, the way of thinking is that the relationship between fast and slow pointer is fast is two times than slow, then we could find the position of the first meet, its distance to the circle start is the same as the distance from list begining to the circle start, thus, we could get the start position of the circle.
